Overview

TFCGT Bridge aims to make TerraFirmaCraft and GregTech CEu behave as a single, cohesive progression: unified materials & recipes, GT-style veins inside TFC geology, KubeJS / JEI support, and optional UI themes.

Why this bridge?

TerraFirmaCraft offers deep geological realism and metallurgy. GregTech brings advanced machines and progression depth. The bridge removes duplication, aligns progression, and gives modpack makers flexible configuration and scripting hooks.

Project Phases (Planned)

All phases are currently planned. Nothing has been marked complete unless explicitly stated in the repository issues.

Phase 1 — Foundation

Unified material registry, ID normalization, base API & datapack format design.

Phase 2 — Worldgen

GT-style vein placement adapted to TFC rock strata; data-driven vein definitions & config.

Phase 3 — Metallurgy

Align melting points, alloys, bloomery ↔ machines, and canonical metal outputs.

Phase 4 — Recipes & Progression

Recipe rewrites, progression gating, and JEI clarity for unlocks.

Phase 5 — Integrations

Native KubeJS & JEI hooks so pack makers don't need extra bridge mods.

Phase 6 — UI & Themes

Optional menu and JEI theming to reflect the mod's brand aesthetic.

Phase Details (Technical)

Phase 1 — Foundation

Goals

  • Provide canonical material keys (e.g., copper, tin, bismuth).
  • Expose Java API for other mods to query canonical items.
  • Design a datapack tag generator for `forge:ingots/*` and similar.

Acceptance Criteria

  • APIs loaded after all mods; no race conditions.
  • Datapack tags write deterministically.
Phase 2 — Worldgen

Goals

  • GT-style veins place only into compatible TFC rocks.
  • Deterministic placement using world seed + chunk pos + vein id.

Implementation Notes

  • Use data-driven `PlacedFeature` where possible.
  • Cache TFC rock column lookups per chunk for performance.
Phase 3 — Metallurgy

Map TFC metallurgy ages to GT tiers (configurable), and provide recipe gating based on progression flags.

Phase 4 — Recipes & Tiering

Generate datapack overrides to canonicalize recipes; provide debug tools for pack makers.

Phase 5 — Integrations

Expose KubeJS events like tfcgt.worldgen.modifyVein() and material registration hooks.
